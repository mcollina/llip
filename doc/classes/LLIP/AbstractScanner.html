Class LLIP::AbstractScanner
In: lib/llip/abstract_scanner.rb
Parent: Object

It‘s the base class of all the scanners. It handles common issues like read a multibyte char or setting up correctly an IO to read from. This class is Abstract because it lacks the next method, which a subclass must add.

UTF8_MB_PATTERN = /[\xc0-\xdf]$|[\xe0-\xef]$|[\xe0-\xef][\x80-\xbf]$/   It‘s used to recognize if a char is multibyte

Attributes

current  [R]  It contains the last token generated by the scanner
current_char  [R]  It contains the current char of the input file, it‘s automatically incremented for every read char. It‘s default is -1. For every $-0 read, it‘s re-initialized to -1.
current_line  [R]  It contains the current line of the input file. It‘s default is -1, and it‘s incremented every $-0 the AbstractScanner reads.
source  [R]  It contains the data scanned by the scanner. It can be nil or an IO. To set it, use scan.

Public Instance methods

next() 
  This method is abstract and it must be implemented by a subclass.

This method must read the chars to produce a Token, set it to current and return it. If a char doesn‘t match the first char of a token, this method should raise LLIPError. If a char matches the first char of a token, but the next ones don‘t, this method should raise an UnvalidTokenError.

scan(IO)
scan(String)

It initializes the scanner to scan an IO. If a String is given, it is automatically converted into a StringIO. It memorize the IO in the source attribute.

Protected Instance methods

read_next()

It reads one char from the source and memorize it in the internal variabile @next_char. It handles multibyte chars correctly. If EOF is reached, it will return nil.
